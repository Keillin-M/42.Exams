allowed functions: write, close, select, poll, socket, accept, listen, send, recv, 
bind, strstr, malloc, realloc, free, calloc, bzero, atoi, sprintf, strlen, exit, 
strcpy, strcat, memsset, htons, htonl

write a program that will listen for a client to connect on a certain port 127.0.0.1 and will let clients talk 
to each other.
This program will take as first argument the port to bind to. If no argument is given, it should write 
in stderr "Wrong number of arguments\n", and exit with status 1.
if a system calls retrns an error before the program starts accepting connections, it should return with 
"Fatal error\n" and exit with status 1.
If you can't allocate memory, it should write in stderr "Fatal error \n" and exit with status 1.
Your program must be non-blocking but clients can be lazy and if they don't read your message you must 
NOT disconnect them.
Your program must not contain #define preprocessors
Your program must only listen to  127.0.0.1
The fd that you will receive will already be set to make recv and send to block if select hasn't been 
called before calling them, but will not block otherwise.

When a client connects to the server:
 - the client will be given an ID. The first client will be given the ID 0, and each new client will 
 receive the ID of the last client + 1.
 - %d will be replaced by this ID
 - A message is sent to all clients that were connected to the server : "server: cliet %d just arrived\n"
 
 Clients must be able to send messages to your program:
  - a message will only be printable characters, no need to check
  - a single message may contain multiple \n
  - when the server receives a message, it must resend it to all other clients : "client %d: " before every line
  
  When a client disconnects from the server, a messge is sent to all clients : "server: client %d just left\n"
  
Memory and FD leaks are forbidden
There is a main.c to help (but it contains forbidden functions (printf) and may do thigs wrong)
Warning : the tester expects to send messages as fast as possible, don't do any unnecessary buffering
hint : use nc to test the program
